# ACL Security Lab

## Objective
Control network traffic using Access Control Lists (ACLs) on a Cisco router.

---

## Scenario
- Sales network: 192.168.10.0/24
- HR network: 192.168.20.0/24

Requirement:
- HR should NOT access Sales
- Sales can access shared services

---

## Key Concept
ACLs filter traffic based on rules:
- Permit (allow)
- Deny (block)

---

## Skills Practiced
- Standard ACL configuration
- Traffic filtering
- Network security enforcement

---

## Outcome
Unauthorized traffic between networks is blocked.
